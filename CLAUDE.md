# CLAUDE.md — DevDeck

## Project description and goal

A personal tech news aggregator for a Software Engineering Tech Lead. It collects articles from RSS feeds, uses AI to summarize/classify/score relevance, stores them in a database, and presents them in a clean web UI. Focus areas: engineering tactics, AI usage in engineering, new processes, industry trends.

**Core constraint: everything must run on free tiers only.**

## Tech stack (and why)

- **Frontend**: React + Vite + TailwindCSS, deployed on Vercel (free tier)
- **Backend/data**: Supabase (free tier) — Postgres + auto-generated API
- **RSS fetching + AI processing**: GitHub Actions with scheduled cron (free, no server needed)
- **AI**: Gemini API (free tier) for summarizing and classifying articles
- **Testing**: Vitest (frontend) + simple tests for scraping scripts
- **CI/CD**: GitHub Actions (lint + tests on every PR; branch protection on `main`)

Do not suggest swapping any of these for a "better" alternative unless explicitly asked — the constraints (free tier, consistency with other personal projects) are deliberate.

## Database schema (Supabase)

`articles` table:

| Column           | Type        | Notes                                       |
|------------------|-------------|----------------------------------------------|
| id               | uuid (PK)   | auto-generated                                |
| title            | text        |                                                |
| url              | text        | unique, to prevent duplicates                 |
| source           | text        | name of the originating feed                  |
| published_at     | timestamptz |                                                |
| summary_ai       | text        | summary generated by Gemini                   |
| category         | text        | e.g. "AI", "Tactics", "Processes", "Industry"  |
| relevance_score  | int         | 1–10, assigned by Gemini                       |
| raw_content      | text        | original content/excerpt, optional             |
| created_at       | timestamptz | default now()                                  |

Unique index on `url` for idempotency.

## Fetch + classification logic

- `fetch-feeds.js` reads `scripts/feeds.json`, fetches each feed, filters out articles already in the DB by `url`
- `classify-with-gemini.js` sends new articles to Gemini asking for JSON-only output:
  ```json
  { "summary": "2-3 sentences", "category": "AI | Tactics | Processes | Industry", "relevance_score": 1-10 }
  ```
- Relevance criteria favors: practical applicability for a tech lead, technical depth, novelty (not generic/marketing content)
- The scheduled Action runs every 6 hours (`cron: '0 */6 * * *'`)
- **All feed URLs must be validated with a real HTTP request before being trusted** — feeds rot over time; treat persistent 404s/empty responses as a signal to find a replacement source, not to silently retry

## Frontend conventions

- Functional components + hooks only, no class components
- Dark mode by default, fully responsive
- Article list sortable by relevance/date/source, filterable by category, with a "score ≥ 7" quick filter

## General conventions

- Scripts in `scripts/` use ESM (`"type": "module"`)
- Never commit API keys — use `.env` locally, GitHub Secrets in CI, Vercel env vars in production
- All UI text in English
- When adding a new RSS source: update `feeds.json`, validate the feed resolves, run the smoke test (see `.claude/skills/add-rss-feed`)
- `npm test` runs Vitest; PRs only merge with green tests (branch protection on `main`)

## Git/GitHub workflow

1. `main` is protected — merges only via PR
2. Branches: `feature/feature-name`
3. CI (`ci.yml`) runs lint + tests on every PR
4. Conventional commits (`feat:`, `fix:`, `chore:`)
5. Automatic deploy on Vercel on every push to `main`
